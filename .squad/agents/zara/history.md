# Zara — Project History

## Project Context

- **Project:** Tollab — Academic management app for Technion students
- **User:** Ibrahim Tabajah
- **Stack:** TypeScript 5.x, Preact, Vite 6.x, Zustand, Firebase v10+, Vitest, Playwright
- **Migration:** Vanilla JS SPA (18 global-scope modules, ~9,800 lines) → modern typed Preact app
- **Hard rules:** No legacy migration code, no compact storage format, pixel-perfect UI fidelity, all work on squad-branch

## Learnings

- **Wave 0 review (PR #46):** Tooling foundation is solid. Strict TS config (`strict: true` + `noUncheckedIndexedAccess`) is the right call for a migration. Vite+Vitest co-located config is clean. CI has all 4 gates (lint, typecheck, test, build) on PRs to squad-branch. Deploy is safely disabled with `if: false`. Legacy `.eslintrc.js` coexists with new flat config — non-blocking but should be cleaned up. All pipeline gates pass locally.
- **Wave 2 review (PR #48):** REQUEST CHANGES. 3-store split (app, profile, UI) is architecturally correct. Storage service is a clean pure data layer with zero store imports. Acyclic import graph — no circular dependencies. Immer used correctly throughout. Found 1 bug: `deleteSemester` reads `state.semesters[idx]` after `splice(idx, 1)`, targeting wrong element for sort-order cleanup. Non-blocking: profile-store couples to app-store via `useAppStore.getState()` in export; selector hooks create new objects on every call (add memoization when components arrive). Pipeline passes (typecheck ✅, lint ✅).
- **Wave 3 review (PR #49):** APPROVE. 10 utility modules with zero cross-cutting imports — each file scoped to a single domain. Strictly acyclic dependency graph (types ← constants ← utils). No store or service imports in utils layer. Barrel exports are explicit named re-exports with type-only syntax where appropriate. Tests mirror source structure 1:1 (8 util test files + 1 validation test file = 206 tests, all passing). Non-blocking: `video.ts` and `ics-parser.ts` define types locally rather than in `@/types`; `generateCourseColor` vs `getNextAvailableHue` are two strategies for the same problem; `substr` deprecated in `string.ts`. Pipeline passes (typecheck ✅, lint ✅, 206/206 tests ✅).
- **Wave 4 review (PR #50):** APPROVE. 8 service files (firebase-config, firebase-auth, firebase-sync, cors-proxy, cheesefork, panopto, youtube, technion-catalog) + existing storage.ts. Strictly acyclic import graph — services depend on types/constants/utils only, zero store or component imports. Firebase properly decomposed: config (init) → auth (Google sign-in) → sync (RTDB push/pull/merge), with auth never leaking into sync (userId passed as parameter). cors-proxy is shared infrastructure for cheesefork/panopto/youtube (valid fan-in pattern). Consistent error handling: graceful null-returns when Firebase unconfigured, custom ProxyFetchError with details, exponential backoff with jitter. Zero legacy/compact format — CloudPayload uses full readable property names. Zero `any` types. Non-blocking: `initAuth`/`onAuthStateChange` are duplicate wrappers; `AppData` interface local to sync could move to `@/types/sync.ts`; `parsePanoptoClipboard` doesn't catch JSON.parse throws. Pipeline passes (typecheck ✅, lint ✅ 0 errors).
